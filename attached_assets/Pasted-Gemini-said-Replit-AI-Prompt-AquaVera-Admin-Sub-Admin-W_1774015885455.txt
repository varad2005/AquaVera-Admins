Replit AI Prompt: AquaVera Admin & Sub-Admin Web Panel
Project Objective
Build a professional, production-grade Admin and Sub-Admin Web Panel for AquaVera, an AI-powered irrigation water request and verification system. The UI must be high-density, data-driven, and follow a "government-grade" agricultural aesthetic.

Core Constraints

Visual Style: Clean, professional, minimalist. Use a forest green and slate gray color palette (#15803d primary).

Typography: Sans-serif (Inter or Roboto).

No Emojis: Absolutely no emojis allowed in the UI; use Lucide-React or Heroicons for all iconography.

Tone: Serious, authoritative, and efficient.

Tech Stack

React (Vite)

Tailwind CSS

Lucide-React (Icons)

React Router DOM (Navigation)

Recharts (Data Visualization)

Shadcn UI (preferred component patterns)

Layout & Navigation

Sidebar: Permanent left sidebar with navigation links: Dashboard, Water Requests, User Management, Activity Logs, and Settings.

Header: Displays current view title, User Role Badge (Admin or Sub-Admin), and Logout button.

Main Content: Scrollable area with a maximum width container for readability.

Feature Requirements & Views

Dashboard (Overview)

Top-row KPI cards: Total Requests, Pending Approvals, Water Volume Allocated (m³), and Flagged Anomalies.

Main Section: A "Request Trends" line chart and a "Crop Distribution" bar chart.

Recent Activity: A vertical feed showing the last 5 system actions.

Water Requests (Table View)

High-density table showing: Request ID, Farmer Name, Crop Type, Duration, Geo-Status (Valid/Invalid), and Status (Pending, Approved, Rejected, Flagged).

Filters: Filter by Status, Crop Type, and Date Range.

Search: Search by Land ID (7/12) or Farmer Name.

Request Detail View (The "Verification" Screen)

Split-screen layout.

Left Side: Farmer Profile (Name, Aadhaar last 4, Land ID) and Request Data (Crop, Duration, Calculated Billing).

Right Side: Verification Panel. Show a "Satellite View" placeholder (Map), the uploaded "Crop Image," and AI Analysis results (Confidence Score: 94%, Location Match: Verified).

Action Bar: Fixed bottom bar with "Approve," "Reject," "Flag for Review," and "Assign to Sub-Admin" buttons.

User Management (Admin Only)

Tabs for Farmers, Sub-Admins, and Admins.

Ability to edit user roles or deactivate accounts.

Activity Logs

Audit trail table: Timestamp, User, Action (e.g., "Approved Request #102"), and IP Address.

Role-Based Access Control (RBAC)

Admin Role: Full access to all views, user management, and system-wide logs.

Sub-Admin Role: Restricted access. Can only view "Assigned" or "Flagged" requests. The "User Management" and "System Logs" links should be hidden or disabled.

Mock Data Requirements

Populate the application with realistic mock data:

Crops: Wheat, Rice, Sugarcane, Cotton.

Status: "Pending Verification," "Satellite Match Confirmed," "Manual Review Required."

Geo-coordinates for location validation simulation.

Component Breakdown

Sidebar.jsx: Navigation with active state styling.

StatCard.jsx: Reusable KPI card with trend indicators.

StatusBadge.jsx: Color-coded badges (Green for Approved, Amber for Flagged, Red for Rejected).

DataTable.jsx: Specialized table component with pagination.

VerificationPortal.jsx: The detailed view for processing requests.
